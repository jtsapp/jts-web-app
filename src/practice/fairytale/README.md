# Fairytale's World — нативная интеграция

Интерактивный движок сказок (бывший standalone `public/practice/fairytales.html`),
встроенный в приложение: карточка сказки на странице Практики открывает мир
полноэкранным оверлеем внутри приложения, с deep-link на конкретную сказку и
возвратом обратно в Практику.

## Файлы

- `engine.js`, `markup.js`, `styles.js` — **сгенерированы автоматически** из
  `public/practice/fairytales.html` скриптом-резаком (см. ниже). Вручную не
  редактировать: правки делаются в HTML-источнике, затем перегенерация.
- `taleWorld.js` — рукописная обёртка: монтирует движок синглтоном, вставляет и
  снимает его глобальные стили, ловит выходы «в библиотеку» и закрывает оверлей.

## Перегенерация после правки fairytales.html

```sh
node scripts/extract-fairytale.js
```

Скрипт режет HTML по границам `<style>/<body>/<script>` (границы захардкожены
номерами строк и защищены sanity-проверками — при изменении структуры HTML
поправить номера в скрипте) и пишет три модуля: `styles.js` = два
`<style>`-блока, `markup.js` = `<body>`, `engine.js` = скрипт, обёрнутый в
`export function createTaleWorld() { … return API }`.

API, которое обёртка ждёт от движка (добавляется `return` в конце фабрики):
`TALES, chooseTale, resumeTale, startOverTale, leaveTale, loadFor, show,
buildLibrary, buildLibLang, refreshContinue`.

## Почему не переписан на React

Движок — ~7600 строк самодостаточного vanilla JS (канвас-частицы, процедурный
SVG-арт, base64-музыка, Web Speech озвучка и микрофон, сохранения в
localStorage). Перепись на React дала бы недели работы и регрессии без выгоды
для пользователя; обёртка сохраняет 100% функциональности, а код живёт в `src/`
и собирается бандлером как обычный lazy-chunk (~3 МБ, грузится при первом
открытии сказки).
