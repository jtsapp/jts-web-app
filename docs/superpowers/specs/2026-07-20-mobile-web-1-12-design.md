## Цель

Мобильная версия сайта, pixel-perfect по Figma-фреймам «iPhone 16 & 17 Pro Max»
1–12. Показывается, когда пользователь заходит с мобильного устройства. Десктоп
не меняется. Результат деплоится на прод (`jts-web-app.vercel.app`), чтобы
проверять с реального телефона.

## Архитектура — подход A (респонсив-CSS)

### Как решается «мобилка»
- Один брейкпоинт: `@media (max-width: 560px)` → мобильный режим. Десктоп ≥561px
  не затрагивается.
- **Без JS-детекции устройства / UA.** CSS media query SSR-безопасен и обходит
  hydration-mismatch (задокументировано в памяти проекта как ловушка).
- `viewport` уже `device-width, initialScale 1` (`src/app/layout.jsx`) — не меняем.

### Стратегия pixel-perfect
- Якорь на дизайн-ширину 440px (iPhone 16 Pro Max в логических px, 1:1 с CSS px).
- Строим **флюидно**: `%`, `clamp()`, `min(x, 100vw)` — «точь-в-точь» на 440px и
  корректное сжатие до 360px. Не жёсткие 440.
- Точные значения (шрифты/цвета/радиусы/отступы) тянутся из Figma-моста поэкранно
  на этапе реализации.

### Мобильный shell (только экраны 10–12)
Единственная структурная добавка. Под брейкпоинтом:
- Десктопный сайдбар 272px (`components/Sidebar.jsx`) → `display:none`.
- Появляется `MobileTopbar` (лого/☰ + карточка профиля «Сакен/Профиль») +
  выезжающий `MobileDrawer` (та же навигация, что в сайдбаре, в оверлее).
- Строится один раз, переиспользуется на всех app-экранах.

### Данные и поток
- Роутинг, i18n, API — без изменений.
- Новые мобильные элементы (маскот на лендинге, топбар, drawer) добавляются
  строго за media query / за флагом мобильного shell — на десктопе их нет.

## Декомпозиция (фазовая реализация)

- **Фаза 0 — фундамент:** брейкпоинт, мобильные дизайн-токены, экспорт ассетов
  (маскот и пр. из Figma), общие мобильные примитивы.
- **Фаза 1 — авторизация (фреймы 1–9):** лендинг, рега-чат, вход, OTP,
  «рега пройдена» + выбор языка.
- **Фаза 2 — shell + вход в приложение (фреймы 10–12):** MobileTopbar + Drawer,
  Обучение, Тьютор-welcome, выбор языка тьютора.

## Проверка

- Каждый экран — в браузер-превью при **440px** (совпадение с Figma) и **360px**
  (проверка сжатия). Скриншот против Figma-фрейма, итеративно до совпадения.
- Регрессия десктопа: те же экраны при 1280px не съехали.
- **Деплой на прод** (`npx vercel --prod`) на границах фаз, не на каждую итерацию,
  чтобы живые юзеры не видели полуфабрикат. Каждый прод-деплой анонсируется
  заранее (публичное действие).

## Вне объёма

- Фреймы 13–18.
- Изменения десктопной вёрстки.
- Реальные голосовые/бэкенд-фичи (используем существующие).
- Отдельный мобильный роут или UA-детекция на сервере.

## Риски / заметки

- Прод использует единственный бэкенд (`dev-server.justtostudy.kz`) — отдельной
  боевой базы у тьютора нет. Деплой на прод безопасен в этом смысле.
- «PT Root UI» через cdnfonts имеет иные метрики, чем в Figma → фикс-заголовки
  верстать явными `\n` + `white-space: pre-line` (ловушка из памяти).
- Экспорт растровых ассетов — через мост `exportAsync` → base64, декодировать
  из tool-results файла (не тащить base64 в контекст).
