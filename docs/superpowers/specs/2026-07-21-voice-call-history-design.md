# Voice call history (list + transcript) — design

**Дата:** 2026-07-21
**Статус:** approved (brainstorming), в реализацию

## Цель

Показать ученику историю голосовых звонков с тьютором: список прошлых
звонков и полный текстовый транскрипт каждого. Сейчас `call_log` не пишет
никто — фича существует только в схеме. Дизайн строго по Figma («Тьютор веб
версия» — desktop, «APP Design mobile» — mobile).

## UX (из Figma)

- **Список** «История разговоров» — в экране «Управление тьютором»
  (`TutorManagePage`). Пункт: заголовок + подпись + время, группировка по дате.
  Примеры: «Практика Present Continious / Практика не пройдена»,
  «Свободный разговор / Общаемся о семье»,
  «Сценарий Job Interview / Успешно пройдено».
- **Транскрипт** — шапка «← Назад | <название>», разделитель даты, пузыри:
  тьютор — фиолетовый слева, ученик — оранжевый справа. Только текст.

## Модель данных

Расширяем существующий `call_log` (сейчас 0 строк, миграция безопасна):

| колонка | тип | назначение |
|---|---|---|
| `transcript` | `jsonb default '[]'` | `[{role:'tutor'\|'learner', text}]` |
| `mode` | `text` | `free` \| `scenario` \| `placement` \| `debate` |
| `scenario_name` | `text` | имя сценария (для заголовка «Сценарий X») |
| `status` | `text` | `passed` \| `failed` \| null (сценарии/практика) |

Уже есть и переиспользуются: `tutor`, `duration_sec`, `created_at` (время/
группировка), `recap` (тема для free-talk — подпись «Общаемся о семье»),
`topics`/`mistakes`/`new_words` (не обязательны для этого экрана).

Заголовок/подпись **не** храним готовой строкой — храним структурно
(`mode`/`scenario_name`/`status`), клиент рисует локализованно (kz/ru/en):
статус `passed`→«Успешно пройдено», `failed`→«Практика не пройдена»; для
free-talk подпись = `recap` (сгенерированный текст как есть). Так экран не
ломается при смене языка UI.

Ключ — `device_id` (аноним и `user-<id>`). `call_log` уже в списке
`REKEY_TABLES` merge.js → анонимные звонки мержатся в аккаунт при входе.

## Захват (агент, cascade)

В конце сессии (в уже существующем shutdown-callback от watchdog):
1. Сериализовать `session.history` → массив реплик `{role, text}`
   (user turns из STT, assistant turns — текст ответа перед TTS).
2. Собрать метаданные: `mode`, `scenario_name`, `status` (из
   `report_task_complete`, если сценарий), `tutor`, `duration_sec`.
3. Один **awaited** POST на `/api/profile/calls` (awaited — shutdown-хвост
   иначе убивается, см. инцидент с fire-and-forget `_bg_tasks`).

Отвергнуто: писать по реплике в реальном времени (N запросов, сложнее). Риск
batch-в-конце: краш до конца → транскрипт потерян; для коротких голосовых ок.

## API

- `POST /api/profile/calls` — служебный ключ `X-Internal-Key`; `device_id` из
  тела через `resolveProfileId` (тот же паттерн, что `log_mistake`). Пишет
  строку `call_log`.
- `GET /api/profile/calls` — список + транскрипт для текущего аккаунта/
  устройства через `resolveProfileId`. Последние ~50, `created_at DESC`.

## Клиент

- Секция «История разговоров» в `TutorManagePage` (desktop + mobile).
- Экран транскрипта (пузыри) — переиспользовать/довязать существующий
  `tutor-chat-history` (в коде есть, навигация не разведена).
- i18n через `useT`/`dict.js` (зона тьютора). Новые ключи: заголовки режимов,
  статусы, «История разговоров», «Назад».
- Стили — в `tutor.css`, классами в существующем стиле; цвета/отступы/радиусы
  строго из Figma.

## Приватность / лимиты

Транскрипт чувствительнее ошибок/слов. Храним по `device_id`, мержим в
аккаунт, отдаём последние ~50. Полной ретенции/удаления в v1 нет (можно
добавить позже).

## Вне scope (v1)

- Удаление/экспорт истории.
- Поиск по транскриптам.
- Транскрипт для gemini-live пути (делаем для cascade — прод-стек).
